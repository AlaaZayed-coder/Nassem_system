# نظام تسعير أصناف النسيم

تطبيق محلي أولاً لإدارة تسعير أكثر من 3,200 صنف لشركة النسيم إخوان. الواجهة عربية بالكامل واتجاهها من اليمين إلى اليسار، والعملة المعروضة هي شيكل.

## التشغيل

1. ثبّت Node.js LTS، ويفضل Node 20 أو Node 22. لا تستخدم Node 24 لهذا المشروع لأن `better-sqlite3` قد يحتاج أدوات بناء إضافية عليه.
2. افتح الملف `start.bat`.
3. عند أول تشغيل سيقوم الملف بتثبيت الحزم، ثم يبني الواجهة، ثم يفتح:

`http://localhost:3000`

يمكن للمطورين تشغيل:

```bash
npm install
npm run dev
```

## الدخول الأولي

كلمة المرور المؤقتة لكل المستخدمين:

`ChangeMe123!`

المستخدمون:

- `ahmad` / أحمد / Admin
- `naseem` / نسيم / Pricing Manager
- `wesam` / وسام / Viewer
- `kareem` / كريم / Viewer

يجب تغيير كلمات المرور لاحقاً قبل الاستخدام الفعلي.

الصلاحيات الأساسية:

- Admin: كل الصلاحيات، إدارة المستخدمين، الاعتماد، القفل وفك القفل.
- Pricing Manager: تعديل واعتماد الأسعار، القفل وفك القفل، التصدير.
- Pricing Editor: إدخال وتعديل وإرسال للمراجعة فقط، بدون اعتماد أو تثبيت.
- Viewer: مشاهدة فقط.

## الاستيراد

من شاشة الأصناف اختر `استيراد CSV`.

قواعد الاستيراد:

- الملف الأصلي: `إدارة الكميات.csv`
- الفاصل: `;`
- الترميز: UTF-8 مع BOM مدعوم.
- يتم استخدام الأعمدة: `الرقم`, `اسم الصنف`, `الوحدة`.
- `item_code` نص دائماً وليس رقماً، لذلك تبقى الأكواد مثل `0416`, `0545A`, `1000/1` كما هي.
- لا يتم استخدام الرصيد أو الكميات في التسعير.
- الصفوف ذات الاسم الفارغ أو `0` أو `1` أو `test` يتم استيرادها ولا تحذف، لكنها توضع في تصنيف `أصناف غير واضحة / تحتاج تنظيف` وحالة `مؤجّل`.
- عند إعادة الاستيراد لا يتم overwrite لأي صنف تم تعديله أو تسعيره أو تثبيته.

## التسعير

- الأسعار تحفظ داخلياً كقروش صحيحة `INTEGER cents` لتجنب أخطاء الكسور.
- السعر المقترح = التكلفة × `(1 + هامش الربح / 100)`.
- التقريب الافتراضي إلى أقرب شيكل.
- لا يوجد VAT ولا ضريبة في نسخة MVP.
- كل الأسعار بدون ضريبة وبعملة شيكل.
- توجد إعدادات `tax_inclusive` و `vat_percent` في شاشة الإعدادات للتحضير المستقبلي فقط، ولا تدخل في أي حساب حالياً.

## الأبواب والتركيب

يمكن تفعيل تسعير الأبواب من شاشة تفاصيل الصنف. يدعم النظام:

- قطعة
- متر مربع
- يدوي

لا يمكن اعتماد صنف باب قبل إدخال سعر بدون تركيب وسعر مع التركيب.

## النسخ الاحتياطي

قاعدة البيانات:

`data/pricing.db`

عند تشغيل السيرفر يتم إنشاء نسخة يومية تلقائياً في:

`backups/`

يمكن أيضاً إنشاء نسخة يدوية من شاشة الإعدادات عبر زر `نسخة احتياطية الآن`.

لا يتم حذف النسخ القديمة تلقائياً في نسخة MVP.

## التصدير

ملفات Excel و CSV تحفظ في:

`exports/`

ويمكن تصدير كل الأصناف أو الفلتر الحالي من شاشة الأصناف.

## بنية المشروع

- `client/` واجهة React + Vite.
- `server/routes/` نقاط API.
- `server/services/` منطق الأعمال مثل التسعير والاستيراد والتصدير والنسخ الاحتياطي.
- `server/repositories/` طبقة الوصول إلى SQLite.
- `server/db/schema.sql` مخطط قاعدة البيانات الموثق للمستقبل.
- `data/pricing.db` قاعدة SQLite المحلية.

تم فصل الواجهة والـ API ومنطق البيانات حتى تكون الهجرة لاحقاً إلى Supabase/PostgreSQL و Vercel أسهل بدون إعادة كتابة التطبيق بالكامل.

## ملاحظات مهمة للهجرة المستقبلية

- جدول `items.item_code` يجب أن يبقى `TEXT PRIMARY KEY`.
- لا تستخدم اسم الصنف كمفتاح لأنه غير فريد.
- لا تحذف الأصناف الأصلية، استخدم الحالة أو `is_active`.
- حقول الضريبة مؤجلة ولا تظهر في واجهة MVP.
